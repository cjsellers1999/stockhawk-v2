# Mission: Design StockHawk database tables

## Why
Design new StockHawk tables confidently by understanding what each existing table owns, how records relate, and where new data belongs.

## Success looks like
- Explain the current schema without reading raw SQL line by line
- Choose whether new data is identity, history, current state, an event, or a read model
- Add primary keys, foreign keys, uniqueness rules, and indexes for the right reasons

## Constraints
- Use stories, diagrams, and concrete StockHawk examples before SQL syntax
- Keep lessons short and reinforce the same mental map over time

## Out of scope
- Advanced PostgreSQL tuning and database administration
