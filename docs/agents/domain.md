# Domain Docs

StockHawk is a single-context repository.

Before working in the codebase:

- Read root `CONTEXT.md`.
- Read relevant architectural decisions under `docs/adr/`.
- Use the glossary's canonical vocabulary.
- Surface conflicts with existing architectural decisions instead of silently overriding them.
- If a referenced domain file does not exist, proceed silently; create domain documentation lazily when decisions are resolved.
